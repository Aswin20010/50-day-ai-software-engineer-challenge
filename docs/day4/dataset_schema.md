# Day 4 — Dataset Schema

## Purpose

This document defines the dataset structure for the research project:

**Automating Enterprise API Development using AI-Driven Requirement and Workflow Synthesis**

The dataset will support two core projects:

1. **Project 1:** Confluence Requirement → Node-RED Flow Generation
2. **Project 2:** Confluence + Node-RED Flow → Excel Requirement Template + Gap Analysis

The goal is to create a structured dataset that can be used to evaluate how accurately AI systems can understand enterprise API requirements, generate workflow logic, and detect implementation gaps.

---

## Dataset Categories

The dataset is divided into four major categories:

```text
datasets/
├── confluence_requirements/
├── nodered_flows/
├── ground_truth/
└── generated_outputs/
```

---

## 1. Confluence Requirement Dataset

### Folder

```text
datasets/confluence_requirements/
```

This folder stores exported requirement documents from Confluence.

Each requirement file should represent one enterprise API requirement.

### Example File Naming

```text
REQ-001-feed-discount.md
REQ-002-interrogate-account.md
REQ-003-customize-fabric-types.md
```

### Required Fields in Each Requirement File

Each requirement document should include:

| Field             | Description                                         |
| ----------------- | --------------------------------------------------- |
| API Name          | Name of the API                                     |
| Endpoint          | REST endpoint path                                  |
| HTTP Method       | GET, POST, PUT, DELETE                              |
| Headers           | Required and optional headers                       |
| Request Body      | Input payload structure                             |
| Response Body     | Expected output payload                             |
| Business Rules    | Validation and orchestration logic                  |
| External Services | Downstream REST/SOAP/DB services                    |
| Database Tables   | PostgreSQL or other persistent tables               |
| Error Handling    | Validation, downstream, business, and system errors |
| Mapping Logic     | Input-to-output or service-to-response mapping      |

---

## 2. Node-RED Flow Dataset

### Folder

```text
datasets/nodered_flows/
```

This folder stores exported Node-RED flows in JSON format.

Each Node-RED flow should correspond to one Confluence requirement.

### Example File Naming

```text
FLOW-001-feed-discount.json
FLOW-002-interrogate-account.json
FLOW-003-customize-fabric-types.json
```

### Required Flow Details

Each Node-RED flow should contain:

| Component            | Description                                     |
| -------------------- | ----------------------------------------------- |
| HTTP In Node         | API entry point                                 |
| Validation Nodes     | Header/body/business validation                 |
| Mapping Nodes        | Request and response transformations            |
| Service Call Nodes   | REST/SOAP/DB integrations                       |
| Error Handling Nodes | Standardized failure handling                   |
| HTTP Response Node   | Final API response                              |
| Subflows             | Reusable wrappers such as SOAP or REST wrappers |

---

## 3. Ground Truth Dataset

### Folder

```text
datasets/ground_truth/
```

This folder stores manually verified correct outputs.

Ground truth files are used to evaluate AI-generated outputs.

### Example File Naming

```text
GT-001-feed-discount.json
GT-002-interrogate-account.json
GT-003-customize-fabric-types.json
```

### Ground Truth Format

Each ground truth file should include:

```json
{
  "requirement_id": "REQ-001",
  "api_name": "FEED Discount",
  "expected_endpoint": {
    "method": "POST",
    "path": "/api/v1/discount/evaluateTransactionDiscount"
  },
  "expected_headers": [],
  "expected_request_fields": [],
  "expected_response_fields": [],
  "expected_business_rules": [],
  "expected_external_services": [],
  "expected_database_tables": [],
  "expected_error_scenarios": [],
  "expected_node_red_nodes": [],
  "expected_excel_sheets": []
}
```

---

## 4. Generated Outputs Dataset

### Folder

```text
datasets/generated_outputs/
```

This folder stores outputs generated by AI systems.

### Examples

```text
generated-node-red-flow.json
generated-excel-specification.xlsx
generated-gap-analysis.md
generated-evaluation-report.md
```

These files will be compared against the ground truth dataset.

---

## Evaluation Use

The dataset will support the following evaluation tasks:

| Task                   | Input                      | Output          | Metric                         |
| ---------------------- | -------------------------- | --------------- | ------------------------------ |
| Requirement Extraction | Confluence document        | Structured JSON | Field-level accuracy           |
| Flow Generation        | Requirement JSON           | Node-RED JSON   | Workflow correctness           |
| Excel Generation       | Requirement + Flow         | Excel workbook  | Sheet completeness             |
| Gap Detection          | Requirement + Flow + Excel | Gap report      | Gap detection precision/recall |

---

## Dataset Quality Rules

Each dataset example should be:

1. Traceable to a real enterprise API requirement.
2. Sanitized to remove sensitive company-specific data.
3. Paired with a manually reviewed ground truth file.
4. Linked to at least one Node-RED flow when available.
5. Suitable for repeatable evaluation.
