## LinkedIn Promotion Draft

I just published the first article from my **50-Day AI Software Engineer Challenge**.

Article title:

**Why AI Code Generation Fails Without Requirement Understanding**

The main idea is simple:

AI can generate code, but enterprise software engineering needs something more important first:

**Structured requirement understanding.**

In enterprise API development, requirements usually do not come in a clean format.

They are often spread across:

* Confluence pages
* Business rules
* Request and response examples
* Downstream API details
* Database mappings
* Error handling expectations
* Engineering discussions

If we directly ask an LLM to generate code from this kind of requirement, the output may look correct but still miss important business logic.

That is why I believe the missing layer is:

```text
Requirement Document
        ↓
Requirement Extraction
        ↓
Workflow Representation
        ↓
Validation
        ↓
Artifact Generation
        ↓
Reviewable Engineering Outputs
```

This article explains why enterprise APIs should be treated as workflows, not just endpoints.

It also explains why AI-assisted software engineering should focus on requirement understanding before code generation.

The larger research direction behind this challenge is:

**Automating Enterprise API Development using AI-Driven Requirement and Workflow Synthesis.**

I will continue sharing progress as I build this challenge.
